# Задание 4. Рефлексия — Шахматные фигуры

WPF-приложение, демонстрирующее использование рефлексии для динамической загрузки библиотеки классов шахматных фигур, создания объектов и вызова их методов через графический интерфейс.

---

## Структура решения


```
Task4Reflection
├── ChessLibrary — библиотека классов (ChessPiece, Queen, Rook, Bishop)
└── ReflectionChessApp — WPF-приложение с рефлексией (MVVM)
```

---

### ChessLibrary

Отдельный модуль (DLL), который загружается динамически через рефлексию.

| Класс | Описание |
|------|--------|
| `IChessPiece` | Интерфейс шахматной фигуры |
| `ChessPiece` | Абстрактный базовый класс (цвет, позиция, метод MoveTo) |
| `Queen` | Ферзь (движение как ладья + слон) |
| `Rook` | Ладья (по строке или столбцу) |
| `Bishop` | Слон (по диагонали) |


---

### ReflectionChessApp (WPF-приложение)

Оконное приложение, реализованное по паттерну MVVM.

| Файл | Назначение |
|------|-----------|
| `MainViewModel.cs` | Загрузка DLL, поиск классов, создание объектов, вызов методов |
| `ObservableObject.cs` | Реализация INotifyPropertyChanged |
| `RelayCommand.cs` | Реализация ICommand |
| `ParameterViewModel.cs` | Ввод параметров конструктора и методов |
| `MethodViewModel.cs` | Отображение методов класса |
| `ClassViewModel.cs` | Представление класса |
| `MainWindow.xaml` | UI |

---

## Сборка и запуск

```bash
dotnet build Task4Reflection.sln
dotnet run --project ReflectionChessApp
```
## Использование

1. Укажите путь к библиотеке `ChessLibrary.dll`:
- ChessLibrary\bin\Debug\net8.0\ChessLibrary.dll

2. Нажмите **«Загрузить библиотеку»**  
   → отобразится список классов (`Queen`, `Rook`, `Bishop`)

3. Выберите класс  
   → появятся его методы и параметры конструктора

---

### Ввод параметров конструктора

Введите значения, например:
- color = Белый
- row = 4
- column = 4


---

### Вызов метода

1. Выберите метод (например `MoveTo`)
2. Введите параметры:
- newRow = 5
-  newColumn = 5


3. Нажмите **«Выполнить»**

---

### Результат

В нижней части окна отобразится:
- результат выполнения метода
- текущее состояние объекта
